# Метод оптимизации параметров видеокодирования для стриминговых сервисов

Репозиторий содержит программные материалы выпускной квалификационной работы на тему:

**«Разработка метода оптимизации параметров видеокодирования для стриминговых сервисов на основе данных, полученных с помощью программно-аппаратного комплекса сбора пользовательских оценок качества видео»**

В репозитории размещены исходные коды программ, использованных для подготовки тестовых видеопоследовательностей, проведения субъективного эксперимента, регистрации пользовательских оценок и последующей обработки результатов.

## Назначение проекта

Цель проекта - построение перцептивно ориентированной лестницы видеокодирования на основе субъективных оценок качества видео.

В отличие от стандартных разрешенческих лестниц, где уровни часто задаются исторически сложившимися форматами отображения, в данной работе уровни формируются по экспериментально измеренным порогам восприятия. Для этого используется зависимость **разрешение(MOS)**, построенная по результатам субъективного эксперимента.

## Программа генерации тестовых видеопоследовательностей

Файл `video_pipeline_v4.py` используется до начала экспериментальных сессий.

Программа выполняет следующие действия:

1. Сканирует каталог с исходными видеофрагментами.
2. Конвертирует исходные файлы формата Y4M в MP4.
3. Формирует четыре тестовые последовательности с различным порядком сцен.
4. Разбивает каждую последовательность на чанки.
5. Для каждого чанка рассчитывает масштаб деградации разрешения.
6. Уменьшает разрешение чанка и масштабирует его обратно до исходного размера кадра.
7. Кодирует чанк с заданными параметрами H.264.
8. Склеивает обработанные чанки в итоговый тестовый стимул.
9. Сохраняет таблицу метаданных `chunks_info.csv`.

Основные параметры задаются в начале файла:

```python
CHUNK_DURATION = 6
CRF = 23
CODEC = 'libx264'
MIN_SCALE = 0.03
INPUT_DIR = r'C:\video_exp'
OUTPUT_DIR = r'C:\video_exp\results'
FFMPEG_PATH = r'C:\ffmpeg\bin\ffmpeg.exe'
```

## Серверное приложение

Файл `server.py` используется во время экспериментальной сессии.

Серверное приложение выполняет следующие функции:

1. Запускает локальный Flask-сервер.
2. Отображает QR-код для подключения участников.
3. Регистрирует участников по индивидуальным номерам.
4. Управляет состоянием экспериментальной сессии.
5. Запускает воспроизведение тестового видео через FFplay.
6. Принимает нажатия участников через веб-интерфейс.
7. Сопоставляет таймкод нажатия с параметрами чанка из `chunks_info.csv`.
8. Сохраняет итоговые результаты в CSV-файл.

Основные маршруты серверного приложения:

| Маршрут | Назначение |
|---|---|
| `/` | Страница участника с кнопкой фиксации оценки |
| `/qr` | Страница с QR-кодом для подключения участников |
| `/register` | Регистрация участника |
| `/state` | Передача текущего состояния эксперимента клиенту |
| `/tap` | Приём нажатия участника и запись оценки |

Для запуска программ требуется:

- Python 3.10 или выше;
- FFmpeg;
- FFplay;
- Flask;
- pandas;
- numpy;
- scipy;
- matplotlib;
- openpyxl;
- ffmpeg-python;
- qrcode.

Установка основных зависимостей:

```bash
pip install flask pandas numpy scipy matplotlib openpyxl ffmpeg-python qrcode pillow
```

FFmpeg и FFplay должны быть установлены отдельно и доступны по путям, указанным в коде.

## Подготовка тестовых видеопоследовательностей

Перед запуском генератора необходимо:

1. Поместить исходные видеофрагменты в каталог `INPUT_DIR`.
2. Проверить путь к `ffmpeg.exe` в переменной `FFMPEG_PATH`.
3. Проверить каталог сохранения результатов `OUTPUT_DIR`.
4. Запустить программу генерации:

```bash
python video_generator/video_pipeline_v4.py
```

После выполнения программы в каталоге результатов должны появиться:

```text
test_stimulus_1.mp4
test_stimulus_2.mp4
test_stimulus_3.mp4
test_stimulus_4.mp4
chunks_info.csv
```

## Проведение экспериментальной сессии

Перед запуском серверного приложения необходимо:

1. Разместить тестовые видеопоследовательности в каталоге `VIDEO_DIR`.
2. Проверить путь к `ffplay.exe` в переменной `FFPLAY_PATH`.
3. Проверить путь к файлу `chunks_info.csv`.
4. Подключить серверный компьютер и смартфоны участников к одной локальной сети.
5. Запустить сервер:

```bash
python server/server.py
```

После запуска сервер выводит локальный адрес. Участники подключаются к нему через браузер смартфона или по QR-коду.

## Формат экспериментальных данных

Итоговая таблица результатов содержит следующие основные поля:

| Поле | Описание |
|---|---|
| `Сессия` | Номер экспериментальной сессии |
| `Испытуемый` | Индивидуальный номер участника |
| `Видео` | Номер тестовой видеопоследовательности |
| `Формат_экрана` | Использованный формат экрана |
| `MOS` | Значение субъективной оценки |
| `Таймкод` | Момент фиксации оценки |
| `Разрешение` | Разрешение чанка в момент нажатия |
| `Масштаб_%` | Масштаб относительно исходного разрешения |
| `Битрейт_кбит` | Битрейт соответствующего чанка |
| `FPS` | Частота кадров |
| `Дата_время` | Дата и время записи результата |

## Обработка данных

Обработка экспериментальных данных включает:

1. Загрузку `qoe_ultimate.csv`.
2. Фильтрацию некорректных записей.
3. Расчёт средних пороговых разрешений для каждого MOS и формата экрана.
4. Расчёт 95% доверительных интервалов среднего по участникам.
5. Построение форматных зависимостей разрешение(MOS).
6. Формирование усреднённой зависимости разрешение(MOS).
7. Построение итоговой перцептивной лестницы кодирования.
8. Сопоставление полученной лестницы с уровнями YouTube и Netflix.

## Результаты

По результатам эксперимента сформирована перцептивно ориентированная лестница кодирования, уровни которой определены на основе субъективно измеренных MOS-порогов. Полученная лестница сопоставлена с техническими уровнями YouTube и кандидатными разрешениями Netflix per-title.

Полное численное сопоставление приведено в файле:

```text
data/comparison_ladders_mos_full.csv
```

## Примечания о воспроизводимости

Для корректного повторения эксперимента необходимо соблюдать те же условия:

- использовать те же тестовые видеопоследовательности;
- применять одинаковые параметры кодирования;
- сохранять структуру чанков и файл `chunks_info.csv`;
- использовать одинаковую логику регистрации оценок;
- проводить обработку данных по тем же скриптам.
